5.4父级元素塌陷的问题

解决方案

  1. 增加父级元素的高度

  2. 增加一个空的div标签，清除浮动

  1. overflow

  2. 父类添加一个伪类 after(推荐使用)

小结：

  1. 浮动元素增加div

    简单，但是空div应该尽量避免

  2. 设置父元素的高度

    简单，但是元素假设有了固定高度，就会被限制

  3. overflow

    简单，但是下拉的一些场景，或者不能被切除的场景不建议使用

  4. 父类添加一个伪类 after(推荐使用)

    写法稍微复杂，但是没有副作用

5.5 对比

  • display
    • 方向不可控
  • float
    • 虽然浮动会脱离标准文档流，所以要解决父级边框塌陷的问题

6.定位

6.1相对定位

相对于原来的位置，进行偏移,相对定位，他仍然在标准文档流中，原来的位置会被保留

6.2绝对定位

定位:基于xxx定位上下左右

  1. 没有父级元素定位的前提下，相对于浏览器定位
  2. 假设父级元素存在定位，我们通常会相对于父级元素进行偏移
  3. 在父级元素范围内移动

相对于父级元素或浏览器的位置，进行指定偏移,绝对定位的话，不在标准文档流中，原来的位置不会被保留
